# Node.js

# 概念

  • 一句话介绍

    • Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时

  • js 运行时，也叫 js 执行环境

    • 编写 js 代码 -> html -> 浏览器
      • 浏览器为什么可以解析一个 网页？
        • UI 渲染引擎
        • JS 执行引擎

  • 主要目的

    • 是为了让 js 在其他地方执行，应对各种业务
      • web 服务开发
        • 游戏服务器
        • 接口服务器
        • …
      • 构建工具
      • 命令行工具
        • npm
        • git
        • …

  • node 相当于将引擎单独封装，所以他可以识别 es 语法部分，但无法识别 DOM 和 BOM

# 版本管理 NVM

# 常用命令

  • nvm 操作
    • 查看版本
      • nvm -v
    • 查看 / 設置根路徑
      • nvm root [path]
  • Node.js 版本操作
    • 查看全部安裝版本
      • nvm ls
    • 查看全部可安裝版本
      • nvm ls-remote
    • 安裝指定版本 Node.js
      • nvm install <version>
    • 卸載指定版本 Node.js
      • nvm uninstall <version>
    • 安裝最新長期支持版本
      • nvm install --lts
    • 切換指定版本 Node.js
      • nvm use node <version>
    • 查看當前使用的 Node.js 版本
      • nvm current

# 概念

  • 全局环境可直接调用的变量

# 常用全局变量

  • console
    • log
      • 日志
    • warn
      • 警告

      • 定义相同的标签，计算标签中间的代码的执行时间

  • export
  • module
  • require()

# 模块和包的概念

# 模块的概念

  • 将一组相关的 API 封装在同一个文件内部，供外界引入使用，这个文件被称为 “模块”

  • 不同的模块有不同的名称

# 包的概念

  • 有些功能的实现，并不能单独的由一个模块实现，需要多个相关模块的配合

  • 把这些相关的模块，放在同一个文件夹内部，再辅助以描述文件，形成了包的概念

  • 描述文件： package.json

    • 属性名 描述 示例
      name 项目名称 “itlike”
      version 版本号（大版本，中版本，小版本） “1.1.0”
      description 项目功能描述 “纯属测试，没啥意义”
      keywords 项目关键词，发布后供用户检索用 [“撩课”, “SZ”, “Node”]
      homepage 项目主页地址 “http://wenhe9.cn ”
      author 项目作者
      main 项目主文件 “index.js”
      scripts 可用脚本 (npm run 名称)
      dependencies 生产环境依赖的包
      devDependencies 开发环境依赖的包 “webpack” : "【4.29.6"】

# node 就遵循了这种模块化的设计

  • 将不同的功能，放在了不同的包 / 模块中
  • 在需要用到的地方，直接引入使用即可

  • 一个模块内的东西，如何导出去让别的模块使用

# 规范方案

# ES6 之前

  • 并没有官方标准的模块化实施方案
  • 在社区中产生了几个主流规范
    • AMD
      • 代表框架 ---- RequireJS
    • CMD
      • 代表框架 ---- SeaJS
    • CommonJS
      • module.exports 导出
      • require() 导入
      • 注意
        • 前端浏览器不支持，在后端 Node 使用

# ES6 之后

  • 之前的主流规范都没有采用
  • 定义了新的规范
    • import
    • export

# CommonJS 模块使用规范

  • 模块作用域
    • 概念
      • 文件作用域
        • Node 中，没法定义自己的全局作用域，只有模块作用域
      • 所以不需要担心一个文件中的变量，会被其他文件修改影响
      • 外部访问不到内部，内部也访问不到外部
  • 模块内向外导出
    • exports 使用
      • 分开导出
        • exports.key = value
      • 统一导出
        • module.exports = 需要导出去的数据
  • 使用方导入模块

# 核心模块如何使用

  • 核心模块为 node 安装时自带的，直接通过 require 引入使用即可

# 常用模块操作

# 路径处理模块 ： path

  • 作用
    • 处理各种路径问题
  • 常用 API
    • 路径的拼接
      • path.join([path1][,path2]...)
    • 路径处理为绝对路径
      • path.resolve([from...,], to)
    • 获取一个路径的拓展名
      • path.extname(p)

# 文件操作模块 fs

  • 作用
    • 可以使用此模块提供的 API 来读写文件
    • 在 fs 模块中，所有的方法都分为同步和异步两种实现
      • 有 sync 后缀的方法为同步方法
      • 不具有 sync 后缀的方法为异步方法
